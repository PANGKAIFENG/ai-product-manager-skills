## 编号策略

- 分类目录使用两位数字前缀，保持 GitHub 阅读顺序稳定。
- 新分类优先追加新编号，不重排既有分类，避免历史链接和目录引用失效。
- 分类目录只做索引，不承载真实 Skill；真实 Skill 目录继续放在仓库根目录。
