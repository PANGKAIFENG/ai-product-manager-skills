# 标准开场 Prompt

## 完整版

遇到重要的复杂问题，可以直接用这个模板开始对话：

```
我接下来会给你一个复杂问题的背景。
请你不要急着给解决方案，也不要默认我的问题定义是对的。

你需要先做 7 件事：
1. 复述你理解的问题（确认我们在讨论同一件事）
2. 找出我表达中的隐含假设（我没有说出来但依赖的前提）
3. 判断这是哪一类问题：已知解 / 设计问题 / 发现问题 / 棘手问题 / 组织问题 / 数据治理问题
4. 判断这个问题在成熟领域里叫什么
5. 挑战我当前思路中最可能错误的地方
6. 给出更高一层的问题定义
7. 告诉我接下来应该先澄清什么，而不是直接进入方案

等这些完成后，我们再讨论解法。
```

## 精简版

当你已经对问题有一定了解但想确认方向时：

```
在我开始设计方案之前，请先帮我：
1. 这个问题在哪个领域有名字？
2. 该领域的标准解是什么？
3. 我当前的思路，最大的假设风险是什么？
```

## 挑战版

当你已经有了方案但想做压力测试时：

```
请不要顺着我的思路补充。
先反驳我，指出最可能错误的 3 个假设。
然后假设 3 个月后这个方案失败了，最可能的原因是什么？
```
