# 10 升级路径

## 适用场景

确定了当前方案后，想规划演进路径。

## 不适用场景

- 一次性任务或无长期演进需求
- 方案还在探索阶段，尚未确定

## Prompt 模板

```
请帮我分析当前方案的演进路径：
①当前方案在什么条件下会触及上限（具体信号是什么）
②那时候的更好方案是什么，为什么比现在的方案好
③我们现在的设计应该为未来的升级留什么接口

补充：升级路径本身也可能被放弃。请额外指出：
如果出现什么新信息，原定的升级路径就不再必要了？
```

## AI 输出要求

1. 当前方案的上限和触达信号
2. 升级后的方案及优势
3. 当前设计应预留的接口
4. 路径放弃信号：什么新信息出现时，升级路径不再必要

## 验收标准

- 上限信号是可观测的（不是"规模变大时"）
- 升级方案和当前方案有明确的差异说明
- 包含"路径放弃信号"——升级路径本身也可以被质疑

## 常见误用

把升级路径当作承诺——路径本身也可能需要放弃，不要被沉没成本绑架。
