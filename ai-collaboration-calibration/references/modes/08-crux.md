# 08 裂缝定位

## 适用场景

方案补丁越来越多，复杂度持续上升。

## 不适用场景

- 问题本身简单、不存在连锁复杂度
- 刚开始设计，还没有积累足够的复杂度来做定位

## Prompt 模板

```
请帮我找这个问题里的 Crux（裂缝）：
标准是——这一点被解决后，其他很多复杂度会自动消失或显著下降。

请输出：
①当前复杂度主要集中在哪里
②哪些复杂度是表象，哪些是根源
③哪个底层点可能是裂缝
④为什么它一旦被解决，其他问题会坍塌
⑤如果它暂时无法解决，当前阶段的次优方案是什么
```

## AI 输出要求

1. 描述当前复杂度分布
2. 区分表象复杂度和根源复杂度
3. 定位裂缝（1-2 个核心点）
4. 说明坍塌机制
5. 给出裂缝无法解决时的次优方案

## 验收标准

- 裂缝是具体的一个点（不是"整体需要重构"）
- 能解释为什么这个点解决后其他问题会简化
- 次优方案和裂缝之间的关系是清晰的

## 常见误用

把根本原因分析和裂缝定位混为一谈。根本原因问"为什么出问题"，裂缝问"解决哪个点影响最大"——是不同的问题。

## 识别裂缝的关键问题

> "如果这一点变了，有多少其他问题会自动消失？"
