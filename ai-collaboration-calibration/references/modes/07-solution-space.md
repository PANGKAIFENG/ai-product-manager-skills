# 07 解法空间扩展

## 适用场景

你已经锁定一个方案，但不确定是否过于狭窄；或感觉被约束卡死，想打开可能性。

## 不适用场景

- 问题已经被分解为明确的执行步骤
- 约束条件已被充分分析且确认无法变动

## Prompt 模板

```
请不要只优化我提出的方案，先列出这个问题的完整解法空间：
- 理想解：不考虑任何约束的最优方案
- 工程可行解：当前资源和平台能力下可以实现的
- MVP解：最小验证版本
- 临时兜底解：先挡住问题，后续再替换
- 不推荐但常见的错误解：别人踩过的坑

然后结合我的资源约束，判断当下应该选哪个，以及哪些路径要保留可切换性。
```

## AI 输出要求

1. 列出 5 种层次的解法
2. 每种解法给出核心逻辑和适用条件
3. 结合用户约束推荐当下选择
4. 说明哪些路径要保留可切换性

## 验收标准

- 解法空间有层次（不是同一层面的 3 个选项）
- 推荐选择有明确的理由（不是"综合考虑"）
- 说明了路径之间的切换条件

## 常见误用

只在用户提出的方案上优化变体，没有真正打开解法空间。注意：规划迁移路径是对的，但要保持"路径本身可以被放弃"的意识。
