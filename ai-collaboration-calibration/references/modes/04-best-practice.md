# 04 最佳实践优先

## 适用场景

在约束内设计方案之前，先建立参照系。

## 不适用场景

- 全新创新方向（无行业标准解可参照时，输出价值有限）
- 问题已经在领域内被充分理解，标准解已知

## Prompt 模板

```
忘掉我们当前的所有约束。如果这类问题在业界有标准解法，那个标准解是什么？
然后回答：
①标准解的核心逻辑是什么
②我们为什么不能直接用（约束是什么）
③我们当前方案在近似标准解的哪个部分
```

## AI 输出要求

1. 描述标准解的核心逻辑
2. 说明约束（为什么不能直接用）
3. 说明当前方案在近似标准解的哪个部分
4. 如果存在多个标准解，说明各自适用条件

## 验收标准

- 标准解是具体的（有名字、有实现方式），不是"用更好的技术"
- 约束分析是具体的（资源、平台、时间），不是"条件不允许"
- 能清晰说明"我们在近似什么"

## 常见误用

找到标准解后直接照搬，跳过约束分析和适配设计。标准解是参照系，不是直接方案。
