# 12 资产化沉淀

## 适用场景

一次重要的探索或讨论结束后，不想只留下结论。

## 不适用场景

- 探索还很浅、没有产生实质性洞察时——硬沉淀只会产出空洞的框架
- 纯执行类任务（翻译、格式化等）

## Prompt 模板

```
请把这次讨论沉淀成四类资产：
①认知资产：我对问题本质的新理解（1-3 条，要求可迁移到其他场景）
②结构资产：可复用的分析框架、判断流程或模型图
③方法论资产：以后遇到类似问题时的操作步骤（写成 checklist 形式）
④Prompt 资产：可直接复用的提问模板或标准开场词

请不要复述过程，要提炼可迁移的方法。
如果这次讨论有值得警惕的反模式（什么坑不要再踩），请单独列出。
```

## AI 输出要求

1. 认知资产：1-3 条可迁移的新理解
2. 结构资产：可复用的框架或流程
3. 方法论资产：checklist 形式的操作步骤
4. Prompt 资产：可直接复用的模板
5. 反模式：值得警惕的坑

## 验收标准

- 资产是可迁移的（能用到其他场景），不是本次讨论的总结
- 方法论是 checklist 形式（可直接执行），不是叙述
- 反模式是具体的（有触发条件和避免方式）

## 常见误用

复述讨论过程而非提炼可迁移的方法，把"过程总结"当成"资产"。

## 资产流水线

```
想法 / 探索 → 快速验证 → 沉淀结构 → 形成资产 → 允许自己离开 → 下次复用
```

一次探索如果没有沉淀为结构、模板、流程或工具，它只是一次灵感消耗。真正的复利来自资产化，而不是探索本身。
