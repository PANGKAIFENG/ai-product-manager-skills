# 03 盲点探测

## 适用场景

你有了框架，但感觉"哪里不对劲"，说不清楚。

## 不适用场景

- 刚开始还没形成框架的阶段——先有框架，才有盲点可探
- 框架描述太模糊，AI 给的"盲点"也会是泛化的

## Prompt 模板

```
我目前的思考框架是：[描述框架]。
站在一个比我更高维的视角，这个框架里哪些重要的东西是隐形的？
请列出 3 个当前框架看不到的盲区，以及每个盲区如果被忽略会带来什么代价。
```

## AI 输出要求

1. 列出 3 个当前框架看不到的盲区
2. 每个盲区说明：为什么在当前框架中隐形、忽略的代价是什么
3. 盲区必须是框架结构性看不到的，不是细节遗漏

## 验收标准

- 盲区是框架层面的，不是执行细节的
- 每个盲区的"代价"是具体的
- 至少有一个盲区让用户产生"确实没想到"的感觉

## 常见误用

描述框架太泛（"我的框架是做产品"），导致输出也泛化。框架描述要具体到结构和假设。
