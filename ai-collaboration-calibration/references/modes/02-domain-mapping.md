# 02 领域定位

## 适用场景

方案越写越复杂，感觉每次都在重新造轮子。用户无法说出这个问题在成熟领域中的名称。

## 不适用场景

- 问题本身全新、无既有领域积累的纯创新场景
- 只是一次轻量表达优化

## Prompt 模板

```
我描述的问题是：[描述现象]。
请判断这个问题在成熟领域里可能叫什么，给出 3 个候选领域术语，并说明：
①每个术语对应的问题边界
②哪个最贴近我的情况
③用这个领域视角看，标准解是什么
④我当前方案与标准解的差距在哪里
```

## AI 输出要求

1. 给出 2-3 个候选领域术语
2. 说明每个术语的边界
3. 判断哪个最贴近当前问题
4. 给出该领域标准解
5. 说明当前方案与标准解的差距

## 验收标准

- 不是泛泛说"这是产品问题 / 技术问题"
- 能说出可检索的领域术语
- 能给出该领域的标准方法
- 能解释为什么这个术语贴近或不贴近

## 常见误用

只让 AI 给方案而不要求命名问题，导致始终在自己的土办法框架里优化。

## 案例

"Stage 3 越来越复杂" → "这叫 Query Relaxation，是 Information Retrieval 的经典问题，行业标准解是 Elasticsearch 的 MUST/SHOULD 评分模型。"
