# AI 输出质量验收标准

## 快速验收表

| 维度 | 好的输出 | 差的输出 |
|------|---------|---------|
| 假设识别 | 指出用户没有明说但依赖的假设 | 只是顺着用户补充细节 |
| 领域定位 | 能说出成熟领域的术语 | 只给泛泛的建议 |
| 问题分类 | 能区分问题类型并说明不同方法 | 直接跳进方案 |
| 解法层次 | 说明理想解、当前解、次优解之间的关系 | 只说"这个方案可行" |
| 演进意识 | 指出当前方案的上限和应切换的条件 | 只给当前阶段做法 |
| 可证伪性 | 能设计验证方式（怎么证伪） | 只给结论，不说怎么验证 |
| 资产沉淀 | 能沉淀可迁移资产 | 只总结本次对话内容 |
| 不确定性 | 主动说明不确定性和知识边界 | 装作全部确定 |

## 核心判断标准

> 好的 AI 协作，不是让你更快拿到答案，而是让你更早发现：自己原来的问题可能问窄了。

## 分项检查清单

### 假设识别质量

- [ ] 假设是真正隐含的（用户没有直接说出来的前提）
- [ ] 不是把用户的显式描述换了个说法
- [ ] 每个假设都有"如果错了会怎样"和"如何验证"

### 领域定位质量

- [ ] 术语是可检索的（能在 Google Scholar / Wikipedia 找到）
- [ ] 不是泛化分类（"产品问题"、"技术问题"不算）
- [ ] 给出了该领域的标准方法，不只是名字

### 问题分类质量

- [ ] 明确了问题类型（6 种之一）
- [ ] 说明了分类对方案设计的含义
- [ ] 分类后选择了对应方法（不是统一用"分析→设计→执行"）

### 方案质量

- [ ] 区分了理想解、当前解、次优解
- [ ] 说明了约束和约束来源
- [ ] 保留了路径切换的可能性
- [ ] 标注了放弃信号

### 不确定性表达

- [ ] 主动说明了哪些判断是高确信的、哪些是不确定的
- [ ] 对不确定的部分给出了验证方式
- [ ] 没有装作所有判断都是确定的
