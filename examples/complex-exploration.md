# Example: complex-exploration

## Use When

You have a complex, ambiguous, multi-round product, strategy, roadmap, pricing, competitive positioning, or methodology task that should not jump directly into a final plan.

## Prompt

```text
$complex-exploration

我想规划一下 Styleclaw 高级版后续 Roadmap。
先不要直接写排期。请先判断这个任务属于什么类型，
原始问题是不是过窄，真正要解决的问题是什么，
以及这次探索应该先产出哪些中间产物。
```

## Expected Output Shape

- Classifies the task type before giving a plan.
- Explains why directly writing a Roadmap would be premature.
- Surfaces hidden assumptions about positioning, system layers, commercial packaging, and competitive advantage.
- Reframes the real question at a higher level.
- Recommends an exploration framework and intermediate outputs.
- Defines what should be researched or validated before final expression.
- Names what can become reusable cognitive, structural, methodological, tool, and influence assets.

## Good Follow-Up

```text
按这个探索框架，先输出第一轮隐含假设表和中间产物规划。
```

or:

```text
现在已经讨论完了，进入 Asset Mode，把这次探索沉淀成下次可复用的方法论和启动 Prompt。
```
