# Security Policy

## Supported Versions

The latest release and the `main` branch receive security and public-safety fixes.

## Reporting a Vulnerability

Do not open a public issue with secrets, private company data, customer data, or exploit details.

Instead, contact the maintainer privately through GitHub profile contact options, or open a minimal public issue that says a private report is needed without including sensitive details.

## Sensitive Content Policy

This repository should not contain:

- API keys, tokens, credentials, or cookies.
- Customer names, customer files, or private business context.
- Internal system names that reveal confidential architecture.
- Proprietary workflow details that are not intended for public release.

If you find sensitive content, report it immediately and avoid cloning, sharing, or quoting the content further.
