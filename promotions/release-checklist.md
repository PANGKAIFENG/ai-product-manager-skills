# Release And Promotion Checklist

## GitHub

- [ ] README first screen explains who this is for.
- [ ] Description and topics are set.
- [ ] LICENSE exists.
- [ ] CONTRIBUTING.md exists.
- [ ] CODE_OF_CONDUCT.md exists.
- [ ] SECURITY.md exists.
- [ ] CHANGELOG.md exists.
- [ ] Issue templates exist.
- [ ] Pull request template exists.
- [ ] `v0.1.0` release is published.
- [ ] Social preview image is uploaded in repository settings.

## Docs

- [ ] Quickstart is readable in 3 minutes.
- [ ] Codex install guide is linked from README.
- [ ] Claude Code install guide is linked from README.
- [ ] Superpowers comparison is linked from README.
- [ ] Each public Skill has an example prompt.

## Promotion

- [ ] Publish Chinese long post.
- [ ] Publish short social post.
- [ ] Share English summary if posting to global communities.
- [ ] Pin the GitHub repository.
- [ ] Ask 3-5 target PM users to try one prompt and report friction.

## After Launch

- [ ] Review issues weekly.
- [ ] Add examples based on real user questions.
- [ ] Track repeated installation friction.
- [ ] Decide whether plugin packaging is needed for v0.2.0.
