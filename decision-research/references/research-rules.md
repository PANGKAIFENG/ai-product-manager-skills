# 调研规则集（R00-R11）

## 启动协议

### R00 — Research Framing Gate

详见 `references/research-framing-gate.md`。

核心：在任何决策锚定之前，先判断用户给的是问题/现象/解法，判断研究层级和类型，必要时上提问题。

失败信号：第一轮搜索就进入具体实现方案。

### R01 — 决策问题锚定

核心：调研服务于决策，不是服务于信息完整性。

操作：AI 把用户描述里隐含的决策问题说出来，同时确认终止条件。

格式：
> 「你问的是 [X]，我理解你的决策是 [具体判断/行动]。这个理解对吗？调研完你需要知道什么才能往下走？」

失败信号：跳过 R01 → 调研结束后用户说「但这不是我想知道的」。

### R02 — 假设显式化

核心：把 AI 当前持有的倾向性假设列出来，预先定义「反转信号」。

操作：在开始搜索前，列出预设并问「如果 [假设 A] 是错的，我应该看到什么信号？」

失败信号：调研结果和用户直觉完全一致，没有任何新发现 → 可能是确认偏差。

### R03 — 暗知识缺口识别

核心：AI 天然只能获取公开的「廉价地表水」，必须主动获取用户的隐性知识。

操作：搜索前必问两个问题：
1. 「你有没有见过把这个做得好的案例，或者已有的参考实现？」
2. 「有没有你已经排除的方向？」

操作规则：
- 拿到参考实现 → 优先深挖，不开启 Web 搜索
- 拿到排除方向 → 记录为「已排除假设」
- 用户说「不知道」→ 正常进入搜索流程

失败信号：调研结束后用户说「其实我们项目里已经有类似实现了」。

## 执行规则

### R04 — 竞争假设枚举（禁止单假设调研）

核心：第一个搜到的结果会无意识成为基准（锚定效应），提前枚举竞争假设打破偏差。

最低要求：至少 2 个互斥假设。

失败信号：整个调研过程只分析了一个方向。

### R05 — 渠道动态选择

核心：不是所有渠道全开，先判断问题类型，再动态选渠道。

来源层级标注（每条信息必须标注）：
- L1：官方文档 / 官方 SDK / 原始文件
- L2：独立第三方研究 / 受信任技术报告
- L3：社区主流实现（依赖数量 + issue 响应速度）
- L4：从业者经验 / 博客 / 二手摘要

失败信号：结论引用的全是 L4 来源，没有任何 L1/L2 交叉验证。

### R06 — 反对证据搜索

核心：主动搜索反对证据，而非支持证据。

诊断性证据标准：
- 有价值的证据：在假设 A 成立时「很可能出现」，在假设 B 成立时「几乎不会出现」
- 无贡献的证据：在所有假设下都「合理存在」

失败信号：找不到任何反例 → 搜索空间太窄。

### R07 — 里程碑推演

核心：不在 V1 阶段跳到 V4 结论。

| 里程碑 | 输入 | 核心问题 | 输出 |
|--------|------|---------|------|
| V1 问题成形 | 用户描述 | 我们实际在问什么 | 可验证的研究问题 |
| V2 假设矩阵 | 竞争假设 | 哪些已有证据 | 假设 × 证据状态表 |
| V3 排除轮 | 诊断性证据 | 哪些假设被排除 | 排除列表 + 理由 |
| V4 残差分析 | 剩余假设 | 置信边界在哪里 | 结论 + 不确定性 |

失败信号：第一轮搜索就直接输出推荐方案。

## 终止规则

### R08 — 三角收敛终止

条件：剩余假设 ≤ 1 个，且有 ≥ 2 个独立来源支持。

「独立」的定义：同一公司的两篇博客不算独立；官方文档 + 独立工程师的实践报告才算独立。

### R09 — 信息饱和终止

条件：连续 3 条新证据在假设矩阵中没有新的排除能力。

### R10 — 暗知识缺口终止

条件：不确定性的核心是情境变量。

操作：宣布调研到达边界，转向用户追问或建议 PoC。

ROI 熔断触发点：调研进行到约 50% 时，如果置信度不足，触发范围评估。

## 输出规则

### R11 — 有立场结论

核心：「给矩阵让用户自选」是调研不够深的表现。

必要结构：推荐 + 置信度 + 排除理由 + 前提 + 颠覆条件 + 待验证。

置信度标签：
- 高：≥2 个独立 L1/L2 来源，无重大反对证据
- 中：有 L1/L2 来源，但存在未解决的竞争假设
- 低：主要来自 L3/L4，或核心假设未验证

Evidence vs Interpretation Split：
- `[Evidence]`：外部公开资料
- `[Local Context]`：本地项目事实
- `[Inference]`：基于证据的推断
- `[Judgment]`：当前产品/战略判断
- `[Open]`：仍未决定
